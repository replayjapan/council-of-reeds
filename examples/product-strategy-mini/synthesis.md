# Synthesis

## What The User Asked For

> I want to know if we should build a very lightweight meeting-notes tool for small internal teams.

> It should not become a full project management system.

## What The Council Agreed On

- Do not build a full project management tool.
- Keep the first version small.
- Core features should be notes, decisions, action items, and search.
- Integrations and complex permissions should be excluded.

## Real Disagreements

- Claude favored a fast prototype.
- ChatGPT wanted stronger validation before building.
- Gemini recommended a discovery step before code.

## PLAI Recommendation

Do one week of discovery before building.

Interview or observe three to five teams. If they can name specific examples of lost decisions or follow-ups, build a narrow prototype.

## First Prototype Scope

- meeting note page
- decisions section
- action items section
- search
- copy/share summary

## What Not To Do

- project boards
- calendar sync
- reminders
- advanced permissions
- broad integrations

## First Action

Collect five real meeting notes and ask the users what they could not find later.
