# Round 1

## Recommendation

Prototype it, but only if the first version is a notes capture and retrieval tool, not a task platform.

## First Version

- meeting title, date, attendees
- notes body
- action items as plain checkboxes
- search across notes
- simple export

## Exclude

- calendar sync
- permissions matrix
- project boards
- reminders
- integrations

## Biggest Risk

The idea becomes too similar to existing note tools unless it has one clear advantage: very fast capture and retrieval for recurring internal meetings.

## Strongest Counter

The market is crowded. If the team cannot name a specific pain that current tools do not solve, it should not build this.
